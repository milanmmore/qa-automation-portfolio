API Testing Basics CheatSheet

Essential API Testing Concepts & Techniques

🔍 What is API Testing?

API Testing ensures APIs meet functionality, reliability, performance, and security requirements.

  • 🧪 Early Testing: Test before UI is ready
  • ⚡ Faster Execution: No browser overhead
  • 📈 Better Coverage: Test business logic directly
  • 🌐 Language Independent: Test any API regardless of implementation

📊 API Types

REST APIs

  • Protocol: HTTP/HTTPS
  • Data Format: JSON, XML
  • Methods: GET, POST, PUT, DELETE, PATCH
  • Stateless: Each request is independent

SOAP APIs

  • Protocol: HTTP, SMTP, TCP
  • Data Format: XML
  • Standards: WSDL, WS-Security
  • Stateful: Can maintain state

GraphQL APIs

  • Protocol: HTTP
  • Data Format: JSON
  • Single Endpoint: One endpoint for all operations
  • Flexible Queries: Clients specify exact data needed

🔧 HTTP Methods (CRUD)

GET - Read

GET /api/users
        GET /api/users/123
        GET /api/users?page=1&limit=10

POST - Create

POST /api/users
        Content-Type: application/json
        
        {
          "name": "John Doe",
          "email": "john@example.com"
        }

PUT - Update (Complete)

PUT /api/users/123
        Content-Type: application/json
        
        {
          "name": "John Doe",
          "email": "john@example.com",
          "age": 30
        }

PATCH - Update (Partial)

PATCH /api/users/123
        Content-Type: application/json
        
        {
          "age": 31
        }

DELETE - Delete

DELETE /api/users/123

🔐 Authentication Methods

 
X-API-Key: your-api-key-here
Authorization: Bearer your-jwt-token-here
Authorization: Basic base64(username:password)
Authorization: Bearer access-token

📋 API Testing Checklist
  • ✅ Valid requests return expected responses
  • ❌ Invalid requests return appropriate error codes
  • 🔍 Required fields are validated
  • 🧪 Optional fields are handled correctly
  • 📐 Data types are validated
  • 📊 Business rules are enforced

🛠️ Testing Tools

Postman

pm.test("Status code is 200", function () {
          pm.response.to.have.status(200);
        });

Python Requests

import requests
        response = requests.get('https://api.example.com/users')
        assert response.status_code == 200

cURL

curl -X GET https://api.example.com/users

📈 API Testing Metrics
  • 📊 Response Time
  • ✅ Success Rate
  • ❌ Error Rate
  • 🕒 Availability

🚀 Best Practices
  • 🧪 Test both positive and negative cases
  • 📐 Validate response structure and data types
  • 🔁 Automate repetitive tests
  • 📚 Document test scenarios clearly
