üß≠ What is Page Object Model?

The Page Object Model (POM) is a design pattern used in test automation to create maintainable, scalable, and readable test suites. Each web page or component is represented by a class that encapsulates its locators and interactions.

üéØ Benefits of POM in Playwright

  • Easy Maintenance: Centralizes locators and actions, simplifying updates.
  • Reusability: Share page objects across multiple tests.
  • Improved Readability: Abstracts complex interactions into clean methods.
  • Reduced Duplication: Encapsulates common actions.
  • Better Debugging: Isolates issues within specific page objects.

üß± Structure of a Page Object

Each page object class typically includes:

  • Locators for elements
  • Methods to interact with those elements
  • Assertions to verify page behavior

üí° Example: Playwright Page Object

// login-page.ts
import { expect, Locator, Page } from '@playwright/test';

export class LoginPage {
  readonly page: Page;
  readonly usernameInput: Locator;
  readonly passwordInput: Locator;
  readonly loginButton: Locator;

  constructor(page: Page) {
    this.page = page;
    this.usernameInput = page.locator('#username');
    this.passwordInput = page.locator('#password');
    this.loginButton = page.locator('button[type="submit"]');
  }

  async goto() {
    await this.page.goto('https://example.com/login');
  }

  async login(username: string, password: string) {
    await this.usernameInput.fill(username);
    await this.passwordInput.fill(password);
    await this.loginButton.click();
  }

  async assertLoginSuccess() {
    await expect(this.page.locator('.welcome')).toBeVisible();
  }
}

// login.spec.ts
import { test } from '@playwright/test';
import { LoginPage } from './login-page';

test('successful login', async ({ page }) => {
  const loginPage = new LoginPage(page);
  await loginPage.goto();
  await loginPage.login('user1', 'securePass');
  await loginPage.assertLoginSuccess();
});
