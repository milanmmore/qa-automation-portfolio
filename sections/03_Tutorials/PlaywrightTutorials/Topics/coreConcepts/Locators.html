üß≠ Introduction

Locators and selectors are the backbone of Playwright‚Äôs automation capabilities. They allow you to find and interact with elements on a web page in a reliable and readable way.

üîç What Are Locators?

Locators are Playwright‚Äôs abstraction over selectors. They:

  • Automatically wait for elements to be ready
  • Retry actions until they succeed
  • Make your tests more stable and readable
const locator = page.locator('text=Login');
await locator.click();

üßµ Types of Selectors in Playwright

1. Text Selector

await page.locator('text=Submit').click();

2. CSS Selector

await page.locator('.btn.primary').click();

3. XPath Selector

await page.locator('//button[text()="Submit"]').click();

4. Role Selector

await page.getByRole('button', { name: 'Submit' }).click();

5. Label Selector

await page.getByLabel('Username').fill('myUser');

6. Placeholder Selector

await page.getByPlaceholder('Enter email').fill('test@example.com');

7. Test ID Selector

<button data-testid="login-button">Login</button>
await page.getByTestId('login-button').click();

üß™ Locator Methods

Method Description
.click() Clicks the element
.fill(value) Fills input fields
.check() Checks a checkbox
.selectOption() Selects a dropdown option
.nth(index) Targets a specific element in a list
.first() / .last() Targets first/last element 
await page.locator('.item').nth(2).click(); // Clicks the third item

‚úÖ Best Practices

  • Use role, label, or test ID selectors for stability.
  • Avoid brittle selectors like deeply nested CSS or XPath.
  • Chain locators for precise targeting:
await page.locator('.form').locator('button:has-text("Submit")').click();

üß† Bonus: Locator Assertions

await expect(page.getByRole('button', { name: 'Submit' })).toBeVisible();
await expect(page.locator('.error')).toHaveText('Invalid credentials');
